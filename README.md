# E-Commerce-Data-End-to-End-Analysis
OVERVIEW

This project presents an end to end data analysis workflow on an ecommerce dataset. The process starts from raw data and moves through data assessment, data cleaning, exploratory data analysis, and final visualization.

The objective is to understand customer behavior, product performance, and operational trends using structured analysis and real world data.


DATASET

The dataset consists of multiple raw CSV files containing information about customers, orders, products, payments, reviews, and logistics.

These files were used in their original form as the foundation of the analysis without heavy preprocessing at the beginning.


DATA ASSESSMENT

In this stage, the dataset was examined to understand its structure and data quality.

Missing values, duplicate records, inconsistent formats, and potential issues were identified to prepare for the cleaning process.


DATA CLEANING

The cleaning process focused on making the dataset consistent and analysis ready.

This includes handling missing values, correcting data types, removing duplicates, and applying necessary transformations.


EXPLORATORY DATA ANALYSIS

Exploratory analysis was performed to uncover patterns and relationships within the data.

Customer purchasing behavior, product category performance, review trends, and delivery patterns were analyzed to derive meaningful insights.


KEY INSIGHTS

Customer activity varies across different time periods with noticeable peaks in order volume

A small group of customers contributes to a significant portion of total revenue

Certain product categories dominate overall sales performance

Delivery delays have a visible impact on customer review scores

Customer preferences differ in terms of payment and ordering patterns


KPIS

Total revenue generated from all orders

Total number of orders

Total number of customers

Average order value

Revenue contribution by product category

Review score distribution

Delivery time and delay patterns


VISUALIZATION

A dashboard with two sheets was created to summarize the findings.

One sheet focuses on overall business performance and customer insights, while the other highlights product and logistics analysis.

The visualization supports the analysis rather than being the primary focus of the project.


PROJECT STRUCTURE

Raw data files include all original CSV datasets used in the project

Data assessment notebook contains initial data inspection and observations

Data cleaning notebook includes all preprocessing steps

EDA notebook presents analysis and insights

Visualization file contains the final dashboard


CONCLUSION

This project demonstrates a complete data analysis workflow built independently from raw data to insights.

It reflects a strong focus on data understanding, cleaning, and exploration to extract meaningful business insights.
